# Jekyll Site Rebuild Guide

## Overview
This guide documents the **CORRECT** way to rebuild your Jekyll al-folio portfolio site using Docker. The site uses a pre-built Docker image that handles all Ruby dependencies and build complexities.

## Quick Rebuild (Use This!)

Run the rebuild script:
```bash
./rebuild_jekyll.sh
```

That's it! Your site will be rebuilt and available at http://localhost:8080

## What the Rebuild Process Does

1. **Pulls the pre-built Docker image** (amirpourmand/al-folio:v0.16.3)
2. **Starts the container** with your site mounted
3. **Installs dependencies** inside the container
4. **Rebuilds the Jekyll site** with all your latest changes
5. **Starts the development server** on port 8080 with auto-rebuild

## Understanding the Architecture

### The Two-Layer System

Your site has two distinct layers:

1. **Source Files** (what you edit):
   - `_casestudies/` - Your case study markdown files
   - `_projects/` - Your project markdown files
   - `_pages/` - Your static pages
   - `_config.yml` - Site configuration
   - `assets/` - Images and other assets

2. **Built Site** (what gets served):
   - `_site/` - The compiled HTML/CSS/JS site
   - Generated by Jekyll from your source files
   - Must be rebuilt when source files change

### Why Docker?

- **Dependency Hell Solved**: Jekyll requires Ruby 2.6+ with specific gem versions
- **Pre-built Image**: Uses amirpourmand/al-folio:v0.16.3 with everything configured
- **Consistency**: Same environment every time, no "works on my machine" issues
- **Isolation**: Doesn't pollute your system with Ruby dependencies

## Manual Rebuild Steps (if script fails)

### Step 1: Stop Any Running Containers
```bash
docker compose down
```

### Step 2: Pull the Latest Image
```bash
docker compose pull
```

### Step 3: Start the Container
```bash
docker compose up -d
```

### Step 4: Install Dependencies Inside Container
```bash
docker exec new-portfolio-jekyll-1 bundle install
```

### Step 5: Rebuild the Site
```bash
docker exec new-portfolio-jekyll-1 bundle exec jekyll build
```

### Step 6: Start the Development Server
```bash
docker exec new-portfolio-jekyll-1 sh -c "bundle exec jekyll serve --watch --port=8080 --host=0.0.0.0 --livereload --verbose --trace" &
```

### Step 7: Verify
Visit http://localhost:8080 to see your rebuilt site.

## Checking Status

### Is Docker Running?
```bash
docker ps
```
You should see `new-portfolio-jekyll-1` in the list.

### Check Rebuild Timestamps
```bash
ls -la _site/ | grep -E "casestudies|projects"
```
Should show today's date/time.

### View Jekyll Server Logs
```bash
docker logs new-portfolio-jekyll-1
```

## Common Issues and Solutions

### Port 8080 Already in Use
```bash
# Find what's using port 8080
lsof -i :8080

# Kill the process (replace PID with actual number)
kill -9 <PID>

# Or stop Docker container
docker compose down
```

### Container Won't Start
```bash
# Remove old containers
docker compose down
docker system prune -f

# Start fresh
./rebuild_jekyll.sh
```

### Changes Not Showing
The site auto-rebuilds, but sometimes needs a manual trigger:
```bash
docker exec new-portfolio-jekyll-1 bundle exec jekyll build
```

### Old Content Still Showing
Your browser might be caching:
1. Hard refresh: Cmd+Shift+R (Mac) or Ctrl+Shift+R (PC)
2. Open in incognito/private window
3. Clear browser cache

## What NOT to Do

❌ **Don't use the Python server on port 8000** - it serves stale content from March 13
```bash
# DON'T DO THIS
python3 -m http.server 8000 --directory _site  # Serves OLD content!
```

❌ **Don't try to build locally with Ruby** - dependency conflicts
```bash
# DON'T DO THIS
bundle install  # Will fail with Ruby version issues
jekyll build    # Won't work without proper dependencies
```

❌ **Don't edit files in `_site/`** - they get overwritten on rebuild

## Auto-Rebuild Feature

Once the Docker container is running with `--watch`, it automatically rebuilds when you:
- Edit markdown files in `_casestudies/`
- Edit markdown files in `_projects/`
- Modify `_config.yml`
- Change any source file

You'll see rebuild messages in the Docker logs:
```bash
docker logs -f new-portfolio-jekyll-1
```

## File Structure Reference

```
new-portfolio/
├── _site/                # ← Built site (auto-generated, don't edit!)
│   └── [HTML files]      # ← Served at http://localhost:8080
├── _casestudies/         # ← Your case study sources (edit these!)
├── _projects/            # ← Your project sources (edit these!)
├── assets/               # ← Images and assets
├── _config.yml           # ← Site configuration
├── docker-compose.yml    # ← Docker configuration
├── Gemfile              # ← Ruby dependencies (Docker handles this)
└── rebuild_jekyll.sh    # ← Rebuild script (run this!)
```

## Summary

1. **To rebuild**: Run `./rebuild_jekyll.sh`
2. **To view**: Open http://localhost:8080
3. **To edit**: Modify files in `_casestudies/`, `_projects/`, etc.
4. **Auto-rebuild**: Happens automatically while Docker is running

Remember: Docker on port 8080 = Fresh content. Python on port 8000 = Stale content from March 13.
